CORBA.ts

A TypeScript implementation of CORBA 3.4 for Deno

Build distributed, interoperable applications with a modern TypeScript-first approach. CORBA.ts provides full CORBA 3.4 specification compliance with Object Request Broker (ORB), GIOP/IIOP protocol support, and comprehensive IDL mapping.

Quick Installation

Get started with CORBA.ts in seconds using JSR or Deno.land:

// Import from JSR (recommended) import * as CORBA from "jsr:@eai/corba"; // Or import from Deno.land import * as CORBA from "https://deno.land/x/corba_ts/mod.ts";

No complex build setup required - just import and start building distributed applications!

Simple Example

Create a CORBA servant and start serving requests:

import { init, ORB_instance, getRootPOA } from "jsr:@eai/corba"; import { Servant } from "jsr:@eai/corba/poa"; // Initialize CORBA runtime await init(); const orb = ORB_instance(); const rootPOA = getRootPOA(); // Create and activate servant class HelloServant extends Servant { sayHello(name: string): string { return `Hello, ${name}!`; } } const servant = new HelloServant(); const oid = await rootPOA.activate_object(servant); await orb.run();

Architecture Overview

Core Components

  • ORB (Object Request Broker) - Central communication hub
  • POA (Portable Object Adapter) - Object lifecycle management
  • TypeCode System - Type information and validation
  • Value Types - Complex data structures

Network Layer

  • GIOP - General Inter-ORB Protocol
  • IIOP - Internet Inter-ORB Protocol
  • CDR - Common Data Representation
  • IOR - Interoperable Object References

Services

  • Naming Service - Object directory
  • DII - Dynamic Invocation Interface
  • IDL Parser - Interface definitions
  • Policy Management - Behavior configuration

Use Cases

Enterprise Integration

Connect modern TypeScript applications with legacy CORBA systems in enterprise environments. CORBA.ts provides seamless interoperability while maintaining type safety and modern development practices.

Distributed Systems

Build scalable distributed applications with location transparency. The ORB handles network communication, load balancing, and fault tolerance automatically.

Microservices

Create type-safe microservice architectures with well-defined interfaces using IDL. CORBA.ts provides robust communication patterns and service discovery mechanisms.

Real-time Systems

Develop real-time applications requiring reliable, low-latency communication between distributed components with precise control over Quality of Service (QoS) parameters.
