Getting Started with CORBA.ts

This guide will help you set up CORBA.ts and create your first distributed application. By the end of this tutorial, you'll have a working CORBA server and client communicating over the network.

Prerequisites

  • Deno 1.37+ - Install Deno
  • Basic TypeScript knowledge - Familiarity with TypeScript syntax and concepts
  • Network access - For downloading dependencies and testing client-server communication

üì¶ Installation

CORBA.ts is available on JSR (JavaScript Registry) and Deno.land. We recommend using JSR for the best experience:

Using JSR (Recommended)

import * as CORBA from "jsr:@eai/corba"; import { Servant } from "jsr:@eai/corba/poa"; import { init, ORB_instance, getRootPOA } from "jsr:@eai/corba";

Using Deno.land

import * as CORBA from "https://deno.land/x/corba_ts/mod.ts"; import { Servant } from "https://deno.land/x/corba_ts/src/poa.ts";

üñ•Ô∏è Creating Your First Server

Let's create a simple "Hello World" CORBA server. Create a new file called server.ts:

// server.ts import { init, ORB_instance, getRootPOA } from "jsr:@eai/corba"; import { Servant } from "jsr:@eai/corba/poa"; // Define the servant class class HelloServant extends Servant { // Every servant must have a repository ID _repository_id(): string { return "IDL:Hello:1.0"; } // Our business method sayHello(name: string): string { console.log(`Server: sayHello called with name: ${name}`); return `Hello, ${name}! Welcome to CORBA.ts`; } // Method to get current server time getServerTime(): string { const now = new Date(); console.log(`Server: getServerTime called`); return now.toISOString(); } } async function startServer() { try { console.log("üöÄ Starting CORBA.ts server..."); // Initialize the CORBA runtime await init(); console.log("‚úÖ CORBA runtime initialized"); // Get the ORB instance const orb = ORB_instance(); console.log("‚úÖ ORB instance obtained"); // Get the Root POA and its manager const rootPOA = getRootPOA(); const poaManager = rootPOA.the_POAManager(); console.log("‚úÖ Root POA obtained"); // Activate the POA Manager await poaManager.activate(); console.log("‚úÖ POA Manager activated"); // Create the servant const servant = new HelloServant(); console.log("‚úÖ Servant created"); // Activate the servant const oid = await rootPOA.activate_object(servant); console.log(`‚úÖ Servant activated with OID: ${oid}`); // Get the object reference const helloRef = await rootPOA.id_to_reference(oid); console.log("‚úÖ Object reference obtained"); // Convert to IOR string for clients to use const helloIOR = await orb.object_to_string(helloRef); console.log("\nüìã Server is ready!"); console.log("üìã Copy this IOR for your client:"); console.log("=".repeat(50)); console.log(helloIOR); console.log("=".repeat(50)); // Save IOR to file for easy client access await Deno.writeTextFile("hello.ior", helloIOR); console.log("üíæ IOR saved to hello.ior"); console.log("\nüèÉ Running ORB... Press Ctrl+C to stop"); // Run the ORB (this blocks until shutdown) await orb.run(); } catch (error) { console.error("‚ùå Server error:", error); Deno.exit(1); } } // Handle graceful shutdown Deno.addSignalListener("SIGINT", () => { console.log("\nüõë Shutting down server..."); Deno.exit(0); }); // Start the server if (import.meta.main) { await startServer(); }

üíª Creating Your First Client

Now let's create a client that connects to our server. Create a new file called client.ts:

// client.ts import { init, ORB_instance } from "jsr:@eai/corba"; // Define the interface for type safety interface Hello { sayHello(name: string): Promise<string>; getServerTime(): Promise<string>; } async function runClient() { try { console.log("üîå Starting CORBA.ts client..."); // Initialize the CORBA runtime await init(); console.log("‚úÖ CORBA runtime initialized"); // Get the ORB instance const orb = ORB_instance(); console.log("‚úÖ ORB instance obtained"); // Read the IOR from file (created by the server) let ior: string; try { ior = await Deno.readTextFile("hello.ior"); console.log("‚úÖ IOR loaded from hello.ior"); } catch (error) { console.error("‚ùå Could not read hello.ior file"); console.error(" Make sure the server is running and has created the IOR file"); Deno.exit(1); } // Convert IOR string to object reference const helloRef = await orb.string_to_object(ior); console.log("‚úÖ Object reference obtained"); // Cast to our Hello interface for type safety const hello = helloRef as unknown as Hello; console.log("\nüìû Making remote calls..."); // Make some remote method calls const greeting = await hello.sayHello("CORBA.ts User"); console.log("üì• Server response:", greeting); const serverTime = await hello.getServerTime(); console.log("üì• Server time:", serverTime); // Make multiple calls to demonstrate the connection console.log("\nüîÑ Making multiple calls..."); for (let i = 1; i <= 3; i++) { const response = await hello.sayHello(`Client Call #${i}`); console.log(`üì• Call ${i} response:`, response); // Wait a bit between calls await new Promise(resolve => setTimeout(resolve, 1000)); } console.log("\n‚úÖ Client completed successfully!"); } catch (error) { console.error("‚ùå Client error:", error); if (error.message?.includes("Connection refused")) { console.error(" üí° Make sure the server is running"); } else if (error.message?.includes("Invalid IOR")) { console.error(" üí° Check that the IOR is valid and the server is accessible"); } Deno.exit(1); } } // Run the client if (import.meta.main) { await runClient(); }

üèÉ Running the Application

Step 1: Start the Server

Open a terminal and run the server:

deno run --allow-net --allow-read --allow-write server.ts

You should see output similar to:

Open a new terminal (keep the server running) and run the client:

You should see output like:

Welcome to CORBA.ts üì• Server time: 2024-01-15T10:30:45.123Z üîÑ Making multiple calls... üì• Call 1 response: Hello, Client Call #1! Understanding What Happened

  1. Server Startup: The server initialized the CORBA runtime, created a servant, and activated it in the POA
  2. IOR Generation: The server generated an IOR (Interoperable Object Reference) containing all the information needed to connect to the object
  3. Client Connection: The client read the IOR, converted it to an object reference, and made remote method calls
  4. Network Communication: All communication happened transparently over GIOP/IIOP protocols
