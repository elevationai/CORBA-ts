CORBA.ts API Reference

Complete API documentation for all CORBA.ts classes, interfaces, types, and functions. The API is organized by functional areas to help you find what you need quickly.

  • init() - Initialize CORBA runtime
  • ORB_instance() - Get ORB singleton
  • object_to_string() - Convert to IOR
  • string_to_object() - Parse IOR

  • getRootPOA() - Access root POA

üöÄ Quick Reference

Common Imports

// Core functionality import { init, ORB_instance, getRootPOA } from "jsr:@eai/corba"; // POA support import { Servant } from "jsr:@eai/corba/poa"; // Types and TypeCodes import { TypeCode, Any, TCKind } from "jsr:@eai/corba/types"; // Naming Service import { NamingContext, NamingContextExt } from "jsr:@eai/corba/naming"; // Dynamic Invocation import { create_request } from "jsr:@eai/corba/dii"; // Exceptions import { SystemException, UserException } from "jsr:@eai/corba/exceptions";

Essential Patterns

// Initialize CORBA await init(); const orb = ORB_instance(); // Create servant class MyServant extends Servant { _repository_id(): string { return "IDL:MyInterface:1.0"; } myMethod(): string { return "Hello World"; } } // Activate servant const rootPOA = getRootPOA(); const servant = new MyServant(); const oid = await rootPOA.activate_object(servant); const objRef = await rootPOA.id_to_reference(oid); // Convert to IOR const ior = await orb.object_to_string(objRef); // Use object reference const obj = await orb.string_to_object(ior); const result = await (obj as any).myMethod();

üìù API Conventions

Naming Conventions

  • Functions: camelCase (e.g., string_to_object)
  • Classes: PascalCase (e.g., TypeCode)
  • Constants: UPPER_CASE (e.g., TCKind.tk_string)
  • Private methods: Prefixed with _ (e.g., _repository_id)

Async Patterns

  • All network operations return Promise<T>
  • Use await for all ORB operations
  • Handle exceptions with try-catch blocks
